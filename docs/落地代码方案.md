# AI 模型后端评测工具落地代码方案

## 概述

### 功能模块

- 评测模块：`m1_arch`、`m2_biz`、`m3_component`、`m4_bugfix`
- 流程模块：Phase 1（生成）+ Phase 2（裁判评分）+ Result（归档）
- 工具模块：Go CLI 初始化目录、初始化记录文件、生成评分 JSON 模板

### 依赖关系

- 运行时依赖：Go 1.22+
- 外部工具依赖：Cursor IDE / Cursor CLI（由评测执行侧提供）
- 本项目代码依赖：Go 标准库

### 设计要点

- 严格对齐方案中的目录结构与命名规范
- 模块输入物料与测试床解耦，方便后续补题
- 记录目录按「模型 × 模块」隔离，避免结果互相污染
- 评分模板保持统一 JSON Schema，便于自动化汇总

## API 层（CLI）

**依赖关系**：依赖 `internal/eval` 提供的应用服务；不直接依赖模块题目实现。

| 方法/命令 | 位置 | 用途 |
| --- | --- | --- |
| `main()` | `cmd/evaluator/main.go` | 解析参数并路由到 prepare / record |
| `-phase prepare` | CLI 参数 | 初始化完整目录骨架 |
| `-phase record` | CLI 参数 | 初始化单模型单模块评测产物模板 |

---

## Integration 层（流程编排）

**依赖关系**：依赖 Layout、Score 模型；被 CLI 调用。

| 方法名 | 位置 | 用途 |
| --- | --- | --- |
| `NewApplication(cfg)` | `internal/eval/app.go` | 构建应用实例 |
| `Prepare(ctx)` | `internal/eval/app.go` | 创建全量目录结构 |
| `InitRecord(ctx, model, module)` | `internal/eval/app.go` | 创建结果文件与初始评分模板 |

---

## Handler 层（参数校验与入口适配）

**依赖关系**：依赖 Integration 层；不做业务逻辑。

| 方法名 | 位置 | 用途 |
| --- | --- | --- |
| `main()` 中参数校验 | `cmd/evaluator/main.go` | 校验 module/model/phase 参数合法性 |
| `IsSupportedModel(model)` | `internal/eval/layout.go` | 限制模型枚举 |
| `IsSupportedModule(module)` | `internal/eval/layout.go` | 限制模块枚举 |

---

## Logic 层（核心规则与文件约定）

**依赖关系**：依赖 Models 层；被 Integration 层调用。

| 方法名 | 位置 | 用途 |
| --- | --- | --- |
| `DefaultDirectories()` | `internal/eval/layout.go` | 输出标准目录树 |
| `ResultFileByModule(module)` | `internal/eval/layout.go` | 输出模块结果文件名 |
| `BuildLogFileByModule(module)` | `internal/eval/layout.go` | 输出模块编译日志名 |
| `TestLogFileByModule(module)` | `internal/eval/layout.go` | 输出模块测试日志名 |
| `WriteFileIfNotExist(path, content)` | `internal/eval/app.go` | 幂等写文件 |

---

## Models 层（数据结构）

**依赖关系**：仅依赖标准库 `encoding/json` 与 `time`；被 Logic/Integration 使用。

| 结构体/方法 | 位置 | 用途 |
| --- | --- | --- |
| `Score` | `internal/eval/score.go` | 评分总结构 |
| `ScoreDetail` | `internal/eval/score.go` | 评分维度结构 |
| `DefaultScoreJSON(...)` | `internal/eval/score.go` | 生成初始 `score.json` 模板 |

---

## DI（依赖注入）

**依赖关系**：由 CLI 注入 Config；通过构造函数组装。

| 注入项 | 位置 | 用途 |
| --- | --- | --- |
| `WorkspaceRoot` | `eval.Config` | 定位项目根目录 |
| `Now func() time.Time` | `eval.Config` | 时间注入，便于测试与复现 |
