# AI 模型后端评测工具落地代码方案（当前实现）

## 概述

当前主流程已经从早期 `cmd/evaluator` 手工初始化，切换到统一命令：

- `cmd/ai_eval`（子命令 `init/run/clear`）

核心链路：

1. `ai_eval init` 创建 `eval_records/<model_dir>/<module>/`
2. `ai_eval run` 调用 Cursor Agent 执行 Phase1 生成代码
3. 运行 `modules/m4_bugfix/tests/run_full_chain.sh` 产出编译/测试日志
4. 再调用 Cursor Agent 执行 Phase3 裁判评分
5. 仅保留最终四个产物文件
6. `ai_eval clear` 清理 `eval_records` 下评测文件

## 当前约束

- 自动化命令 `ai_eval` 当前仅支持 `m4`/`m4_bugfix`。
- 阶段日志统一为 `Init -> Phase1 -> Phase2 -> Phase3 -> Done`。
- 仅保留最终产物：
  - `m4_result.go`
  - `m4_build.log`
  - `m4_test.log`
  - `score.json`
