# M4（缺陷修复）自动化评测 SOP（Cursor）

## 目标

通过两条命令完成 M4 评测：

1. `ai_eval_init`：按候选模型初始化目录
2. `ai_eval`：一键执行 `Phase1 -> Phase2 -> Phase3` 并输出最终结果

---

## 待测模型

第一批模型建议：

- `gemini-3.1-pro`
- `gpt-5.3-codex`
- `opus-4.6-thinking`
- `kimi-k2.5`
- `qwen3.5`

完整可选模型见：`docs/模型清单.md`

---

## 前置条件

```bash
cd /Users/zhengyu.hu/Desktop/ai_eval
cursor-agent login
go install ./cmd/ai_eval_init
go install ./cmd/ai_eval
```

---

## Step 1：初始化候选模型目录（仅目录）

```bash
ai_eval_init --models "gemini-3.1-pro,gpt-5.3-codex,opus-4.6-thinking,kimi-k2.5,qwen3.5" --modules "m4"
```

说明：

- 该命令只创建 `eval_records/<model_dir>/m4_bugfix/` 目录
- 不会预写 `go/json/log` 文件

---

## Step 2：执行单模型自动评测

```bash
ai_eval --module m4 --model "opus-4.6-thinking" --judge-model "opus-4.6"
```

---

## 执行过程可观测性

终端会持续输出阶段日志（首字母大写，且无小数阶段名）：

- `Init`
- `Phase1`（代码生成）
- `Phase2`（编译+测试）
- `Phase3`（裁判评分）
- `Done`

长耗时步骤会周期输出：

- `Still running... elapsed=...`

---

## 结果产物（仅保留最终结果）

评测结束后仅保留以下 4 个文件：

- `eval_records/<model_dir>/m4_bugfix/m4_result.go`
- `eval_records/<model_dir>/m4_bugfix/m4_build.log`
- `eval_records/<model_dir>/m4_bugfix/m4_test.log`
- `eval_records/<model_dir>/m4_bugfix/score.json`

中间产物会自动清理，不会保留以免影响后续编译。

---

## `score.json` 的运行时字段

`score.json` 会包含并自动写入：

- `judge_model`
- `runtime_metrics.phase1_seconds`
- `runtime_metrics.phase2_seconds`
- `runtime_metrics.phase3_seconds`
- `runtime_metrics.total_seconds`

其中运行时长会参与裁判评分参考（见 `modules/m4_bugfix/JUDGE_AGENT.md`）。
