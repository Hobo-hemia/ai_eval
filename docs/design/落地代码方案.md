# AI 模型后端评测工具落地代码方案（当前实现）

## 概述

当前主流程切换为统一命令：

- 根入口：`main.go`
- 命令实现：`cmd/init.go`、`cmd/run.go`、`cmd/clear.go`

核心链路：

1. `ai_eval init` 创建 `eval_records/<model_dir>/<module>/`
2. `ai_eval run` 调用 Cursor Agent 执行 Phase1 生成代码
3. 运行模块对应 harness 脚本（如 `modules/m1_arch/tests/run_full_chain.sh`、`modules/m2_biz/tests/run_full_chain.sh`、`modules/m3_component/tests/run_full_chain.sh`、`modules/m4_bugfix/tests/run_full_chain.sh`）产出编译/测试日志
4. 再调用 Cursor Agent 执行 Phase3 裁判评分
5. 仅保留最终四个产物文件
6. `ai_eval clear` 清理 `eval_records` 下评测文件

## 代码层次

### API

| 方法/命令 | 位置 | 用途 |
| --- | --- | --- |
| `main()` | `main.go` | 程序总入口，分发到 `cmd` 包 |
| `Execute(args)` | `cmd/run.go` | 命令路由（init/run/clear） |
| `RunAutoEvaluation(...)` | `internal/internal.go` | internal 对外统一 workflow 入口 |
| `NewApplication(...)` | `internal/internal.go` | 兼容 prepare/record 的轻量入口 |

### Integration（依赖 `internal/module`）

| 方法名 | 位置 | 用途 |
| --- | --- | --- |
| `RunAutoEvaluation(...)` | `internal/workflow/run.go` | 统一编排 Init/Phase1/Phase2/Phase3 |
| `runPhase1(...)` | `internal/workflow/phase1.go` | 隔离工作区代码生成与落盘 |
| `runHarnessByModule(...)` | `internal/workflow/phase2.go` | 执行模块构建/测试脚本 |
| `runJudgePhase(...)` | `internal/workflow/judge.go` | 裁判评分与 score 归一化 |
| `phase1PromptByModule(...)` | `internal/workflow/prompt.go` | 模块化提示词装配 |
| `preparePhase1Workspace(...)` | `internal/workflow/isolation.go` | Phase1 可见性隔离 |

### Logic（`internal/module`）

| 方法名 | 位置 | 用途 |
| --- | --- | --- |
| `ModelDirName(...)` | `internal/module/model_path.go` | 模型名安全化为目录名 |
| `ResultFileByModule(module)` | `internal/module/layout.go` | 模块结果文件名 |
| `BuildLogFileByModule(module)` | `internal/module/layout.go` | 模块编译日志名 |
| `TestLogFileByModule(module)` | `internal/module/layout.go` | 模块测试日志名 |

### Models（`internal/module`）

| 结构体 | 位置 | 用途 |
| --- | --- | --- |
| `Score` | `internal/module/score.go` | 最终评分结构 |
| `RuntimeMetrics` | `internal/module/score.go` | 各阶段耗时指标 |

## 当前约束

- 自动化命令 `ai_eval` 当前支持 `m1`/`m1_arch`、`m2`/`m2_biz`、`m3`/`m3_component`、`m4`/`m4_bugfix`。
- 阶段日志统一为 `Init -> Phase1 -> Phase2 -> Phase3 -> Done`。
- 仅保留最终产物：
  - `<module>_result.go`（M1 为 `m1_result.proto`）
  - `<module>_build.log`
  - `<module>_test.log`
  - `score.json`
