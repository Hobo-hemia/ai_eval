# M2（业务实现）自动化评测 SOP（Cursor）

## 目标

通过统一命令完成 M2 评测：

1. `ai_eval init`：初始化候选模型目录
2. `ai_eval run`：执行 `Phase1 -> Phase2 -> Phase3`
3. `ai_eval clear`：清理评测产物

---

## 评测目标（M2）

M2 聚焦“协议驱动的复杂业务实现”能力，重点验证模型是否能根据 `api.proto` 规划，生成包含：

- 跨服务调用（Pricing / Inventory）
- MySQL 事务
- Kafka 发布
- Redis 幂等与重试标记

且在任意依赖错误下保持业务正确性与数据一致性。

---

## 前置条件

```bash
cd /path/to/ai_eval
cursor-agent login
go install .
```

---

## Step 1：初始化目录

```bash
ai_eval init --models "gpt-5.2-codex,sonnet-4.5" --modules "m2"
```

---

## Step 2：执行单模型评测

```bash
ai_eval run --module m2 --model "gpt-5.2-codex" --judge-model "gemini-3-flash"
```

说明：
- `Phase1` 仅可见 `input/` + `.cursorrules`，不可见 `tests/harness` 与 `JUDGE_AGENT.md`。
- 代码生成完成后才落盘 `eval_records/<model_dir>/m2_biz/m2_result.go` 并进入测试与裁判。

---

## 结果产物

- `eval_records/<model_dir>/m2_biz/m2_result.go`
- `eval_records/<model_dir>/m2_biz/m2_build.log`
- `eval_records/<model_dir>/m2_biz/m2_test.log`
