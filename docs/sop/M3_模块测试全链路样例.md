# M3（组件设计）自动化评测 SOP（Cursor）

## 目标

通过统一命令完成 M3 评测：

1. `ai_eval init`：按候选模型初始化目录
2. `ai_eval run`：一键执行 `Phase1 -> Phase2 -> Phase3`
3. `ai_eval clear`：评测结束后一键清理产物

---

## 评测目标（M3）

M3 聚焦“高抽象、高复用、高并发”的组件能力，要求模型实现可复用的泛型分片缓存组件，并通过并发合同测试与 benchmark。

---

## 前置条件

```bash
cd /path/to/ai_eval
cursor-agent login
go install .
```

---

## Step 1：初始化候选模型目录

```bash
ai_eval init --models "gpt-5.2-codex,sonnet-4.5,gpt-5.3-codex-low-fast" --modules "m3"
```

---

## Step 2：执行单模型自动评测

```bash
ai_eval run --module m3 --model "gpt-5.2-codex" --judge-model "gemini-3-flash"
```

说明：
- `Phase1` 仅加载当前模块输入材料（`input/` + `.cursorrules`），不会向被测模型暴露 `tests/harness` 或 `JUDGE_AGENT.md`。
- 仅在代码生成完成后，结果才会写入 `eval_records/<model_dir>/m3_component/m3_result.go` 并继续后续评测。

---

## 可观测性

终端会输出阶段日志：

- `Init`
- `Phase1`（代码生成）
- `Phase2`（构建+并发测试+benchmark）
- `Phase3`（裁判评分）
- `Done`

长耗时阶段会持续输出：

- `Still running... elapsed=...`

---

## 结果产物（仅保留最终结果）

- `eval_records/<model_dir>/m3_component/m3_result.go`
- `eval_records/<model_dir>/m3_component/m3_build.log`
- `eval_records/<model_dir>/m3_component/m3_test.log`
- `eval_records/<model_dir>/m3_component/score.json`

---

## 清理评测现场

```bash
ai_eval clear
```
