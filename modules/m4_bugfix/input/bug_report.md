# M4 缺陷报告：结算系统神秘连环故障

## 故障背景与现象观察

我们的核心结算系统（Settlement Service）在过去的一个月中，遇到了一系列诡异的生产故障。该系统主要职责是：接收上游系统的日间交易入账，累加每日交易额，当累计金额突破特定的“合规风险阈值”时，触发下游风控通知系统。

然而，在每天下午的高并发结算期，监控系统观察到了以下“灾难性”现象：

1. **诡异的账户余额跳变**：有大商户反馈，明明他们的交易额都是正数进账，但系统后台显示的今日总金额偶尔会变成一个极其离谱的负数。
2. **P0 级系统卡死**：在某些时刻，整个结算服务的所有请求处理全部阻塞（pprof 显示海量 goroutine 卡在获取锁的地方），直接将系统拉跨。
3. **下游风控遭遇“告警风暴”**：风控团队抱怨，一旦某个商户触发了合规阈值，他们就会在接下来的 1 小时内收到几百条一模一样的风险告警，导致风控系统被打挂。
4. **风险事件静默丢失**：与告警风暴相反的是，有时明明触发了阈值，且由于网络抖动导致发送告警失败了，但我们的系统再也没有尝试过重发，导致“漏报合规事件”而面临监管罚款。
