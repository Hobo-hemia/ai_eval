# 客户服务中心 2.0 产品需求（PRD 摘要）

## 背景

我们当前的“在线工单”接口以客服内部视角设计，已经无法支撑多业务线统一接入。  
接下来要升级为“客户服务中心 2.0”，让 App、Web、企业后台都能使用同一套能力。

现有协议文件见：`api.proto`（v1）。

## 本期功能点（产品语言）

1. **统一入口提交诉求**  
   用户不再区分“投诉/咨询/退款”入口，统一提交“服务诉求”，系统按场景自动分发。

2. **状态时间线可视化**  
   前端需要展示每个诉求的处理时间线（受理、转派、处理、关闭、重开）。

3. **协作人机制**  
   一个诉求可能由多个角色共同处理（客服、商家、风控），需要在协议层支持协作人信息。

4. **批量操作**  
   运营需要对一批诉求执行统一动作（批量转派、批量加急、批量关闭）。

5. **SLA 预警能力**  
   前端要能识别“即将超时”的诉求，支持在查询接口中直接返回风险提示字段。

## 对协议改造的明确期望

- 不是简单新增 5 个接口。
- 需要从后端视角抽象“核心链路”，将能力收敛为更少的接口（新增接口 < 5）。
- 允许对 v1 的接口名、请求/响应字段进行重命名与删改。
- 需要兼顾后续扩展性（不同业务线复用同一协议）。

## 非功能要求

- 保持 `proto3`。
- 命名清晰，避免业务方术语歧义。
- 字段演进需考虑兼容（避免无意义字段堆叠）。
